# 项目概览

## 1. 目标

构建一个 **AI 驱动的岗位定制简历生成平台**，帮助用户维护一份长期复用的主档案，系统根据目标岗位自动生成匹配度更高的定制化简历。

## 2. 核心产品定义

> 用户维护一份长期复用的主档案，系统根据目标岗位自动生成更匹配该岗位的定制化简历，支持在线编辑、公开链接分享和 PDF 导出。

## 3. 业务主线

```
主档案管理 → 岗位输入 → 岗位解析 → AI 定制简历生成 → 在线编辑 → 保存草稿 → 发布 → 公开查看 → PDF 导出
```

## 4. 一期 MVP 范围

### 4.1 必须包含

| 序号 | 功能模块 | 说明 |
|------|----------|------|
| 1 | 用户注册/登录 | JWT 鉴权 |
| 2 | 主档案管理 | 基本信息、教育、工作、项目、技能、证书 |
| 3 | 岗位信息输入 | 支持链接或 JD 文本 |
| 4 | 岗位解析 | AI 解析 JD 为结构化数据 |
| 5 | AI 生成定制简历 | 基于主档案和岗位要求生成 |
| 6 | 在线编辑草稿 | 手动调整简历内容 |
| 7 | 保存版本 | 草稿/已发布状态管理 |
| 8 | 发布简历 | 生成公开链接 |
| 9 | 公共链接访问 | 网页查看简历 |
| 10 | PDF 导出 | A4 格式下载 |
| 11 | AI 任务日志 | 记录 AI 调用过程 |
| 12 | 版本复制 | 基于旧版本创建新草稿 |
| 13 | 错误处理与重试 | 基础容错机制 |

### 4.2 一期不做

- 企业招聘端
- 自动投递
- 多人协作
- 高级会员系统
- 模板商城
- 国际化复杂多语言
- 大型后台运营系统
- 高级数据分析看板

## 5. 核心数据实体

| 实体 | 说明 |
|------|------|
| User | 用户账号 |
| ResumeProfile | 主档案 |
| EducationRecord | 教育经历 |
| WorkExperience | 工作经历 |
| ProjectExperience | 项目经历 |
| SkillRecord | 技能 |
| CertificateRecord | 证书/奖项 |
| JobTarget | 目标岗位 |
| ResumeVersion | 简历版本 |
| ResumePublishRecord | 发布记录 |
| AITaskLog | AI 任务日志 |

## 7. 项目约束

### 7.1 AI 集成原则

1. 前端不能直接调用模型 API
2. 所有 AI 调用必须通过后端统一封装
3. AI 输出必须结构化 JSON
4. AI 不能直接输出最终 HTML 作为核心数据
5. AI 只能基于用户提供的事实进行重写
6. 所有关键 AI 输出必须进行 schema 校验
7. AI 调用必须记录日志
8. 必须预留更换模型提供方的能力

### 7.2 代码质量要求

- 命名规范统一
- 分层清晰
- TypeScript 类型完整
- 注释适量
- 便于后续维护
